# Chapter 15: The Yoneda Lemma — 요네다 보조정리

> **핵심 개념**: **요네다 보조정리(Yoneda Lemma)**는 카테고리 이론에서 가장 중요한 결과 중 하나다. 홈 펑터 `C(a, -)`에서 임의의 펑터 F로의 **자연 변환의 집합**이 `F(a)`와 **동형**임을 말한다. 이 정리는 대상을 그 대상과의 관계(사상)로 완전히 특징지을 수 있다는 깊은 통찰을 담고 있다.

---

## 요네다 보조정리의 진술

카테고리 C에서 Set으로의 펑터 F와 C의 대상 a에 대해:

```
Nat(C(a, -), F) ≅ F(a)
```

**"홈 펑터 C(a,-)에서 F로의 자연 변환들의 집합은 F(a)와 동형이다."**

이 동형은 a에 대해 **자연적(natural)**이다.

---

## 직관적 이해

자연 변환 `α :: C(a, -) → F`를 정의하려면 **모든 대상 x**에 대해 성분 `α_x :: C(a,x) → F(x)`를 정해야 한다. 무한히 많은 성분을 정해야 하는 것 같지만, 요네다 보조정리는 **하나의 값** `α_a(id_a) ∈ F(a)`만 정하면 나머지가 자연성 조건에 의해 **자동으로 결정**된다고 말한다.

왜 그런가? x = a일 때 `α_a :: C(a,a) → F(a)`에서 `id_a ∈ C(a,a)`를 대입한 값 `p = α_a(id_a) ∈ F(a)`를 정하면, 임의의 `f :: a → x`에 대해:

```
α_x(f) = (F f)(p)    -- 자연성 조건에서 유도!
```

자연성 조건 `F f ∘ α_a = α_x ∘ C(a, f)`에서 id_a를 대입하면:

```
(F f)(α_a(id_a)) = α_x(C(a,f)(id_a))
(F f)(p) = α_x(f ∘ id_a)
(F f)(p) = α_x(f)
```

따라서 **p를 고르면 전체 자연 변환이 결정**된다.

## Haskell에서의 요네다 보조정리

```haskell
-- 요네다 보조정리:
-- (forall x. (a -> x) -> f x) ≅ f a

-- 방향 1: 자연 변환에서 값으로
yoneda :: (forall x. (a -> x) -> f x) -> f a
yoneda alpha = alpha id    -- id_a를 대입

-- 방향 2: 값에서 자연 변환으로
yonedaInv :: Functor f => f a -> (forall x. (a -> x) -> f x)
yonedaInv fa h = fmap h fa    -- F h를 적용
```

### 구체적 예시: F = List

```haskell
-- Nat(C(a, -), []) ≅ [a]

-- 자연 변환 alpha :: forall x. (a -> x) -> [x]가 주어지면
-- alpha id :: [a]를 얻는다

-- 반대로 [a] 값이 주어지면
-- alpha h = fmap h [a]  로 자연 변환을 복원

-- 예: as = [1, 2, 3] :: [Int]
-- alpha h = fmap h [1, 2, 3] = [h 1, h 2, h 3]
```

### 구체적 예시: F = Maybe

```haskell
-- Nat(C(a, -), Maybe) ≅ Maybe a

-- Maybe a의 값이 Nothing이면: alpha h = Nothing (모든 h에 대해)
-- Maybe a의 값이 Just x이면: alpha h = Just (h x)
```

## 요네다 보조정리의 의미

### 1. 대상을 사상으로 특징짓는다

`C(a, -)` ≅ `C(b, -)` (자연 동형)이면 a ≅ b (동형)이다. 즉, **대상은 그 대상에서 나가는 사상들로 완전히 결정**된다. 대상의 "정체성"은 내부 구조가 아니라 다른 대상과의 관계로 결정된다.

### 2. 무료 정리의 일반화

매개변수적 다형 함수 `forall x. (a -> x) -> f x`의 구현이 `f a`의 원소와 1:1 대응한다는 것은, 타입만으로 구현을 크게 제약할 수 있다는 "무료 정리"의 정수다.

### 3. CPS 변환

`a ≅ forall x. (a -> x) -> x`는 **Continuation Passing Style(CPS)**과 관련된다. 값 a를 직접 다루는 대신, "a를 받는 연속(continuation)"을 다룰 수 있다.

---

## 반변 요네다 보조정리

반변 홈 펑터 `C(-, a)`에 대한 요네다:

```
Nat(C(-, a), F) ≅ F(a)    -- F: C^op → Set (반변 펑터)
```

Haskell에서:

```haskell
-- (forall x. (x -> a) -> f x) ≅ f a   (f는 반변 펑터)
```

---

## 정리

- **요네다 보조정리**: `Nat(C(a,-), F) ≅ F(a)` — 홈 펑터에서 F로의 자연 변환은 F(a)의 원소와 1:1 대응
- **핵심 메커니즘**: `id_a`에서의 값 하나가 전체 자연 변환을 결정 (`α_x(f) = (Ff)(p)`)
- **Haskell**: `(forall x. (a -> x) -> f x) ≅ f a`
- **의미**: 대상은 사상으로 완전히 특징지어진다 (대상의 외부적 정의)
- **응용**: CPS 변환, 최적화 (fusion), 표현 가능 펑터의 판별
- **반변 버전**: `Nat(C(-,a), F) ≅ F(a)` (반변 펑터에 대해)
