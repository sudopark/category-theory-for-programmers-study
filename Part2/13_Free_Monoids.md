# Chapter 13: Free Monoids — 자유 모노이드

> **핵심 개념**: **자유 구성(free construction)**은 최소한의 제약만으로 대수적 구조를 만드는 방법이다. **자유 모노이드(free monoid)**는 집합의 원소들을 "생성원"으로 사용하여, 결합법칙과 단위 원소 외에 추가 등식 없이 만든 모노이드다. Haskell에서 리스트 `[a]`가 바로 타입 a의 자유 모노이드이며, 이 구성은 **망각 펑터(forgetful functor)**의 왼쪽 수반(left adjoint)으로 이해된다.

---

## 자유 구성이란?

집합 `{a, b}`에서 모노이드를 만들고 싶다. 최소한의 규칙만 적용하면:

1. **단위 원소** `e`를 추가
2. 모든 원소의 **쌍(pair)**을 새 원소로 추가: (a,b), (b,a), (a,a), (b,b)
3. 단위 원소와의 쌍은 원래 원소로 식별: (a,e) = a, (e,b) = b
4. 이 과정을 무한히 반복: ((a,b), a), (a, (b,b)), ...
5. **결합법칙**: ((a,b), c) = (a, (b,c))으로 식별

결과적으로 생성원 a, b의 모든 유한 **열(sequence)**이 만들어진다:

```
{e, a, b, ab, ba, aa, bb, aba, bab, aab, ...}
```

이것이 **자유 모노이드**다. 결합법칙과 단위 원소 외에 어떤 추가 등식도 부과하지 않았다 (예: ab = ba 같은 교환법칙 없음).

---

## Haskell에서의 자유 모노이드

Haskell에서 타입 `a`의 자유 모노이드는 **리스트 `[a]`**이다:

```haskell
-- 리스트 = 자유 모노이드
-- 이항 연산 = 연결 (++)
-- 단위 원소 = 빈 리스트 []

instance Monoid [a] where
    mempty  = []
    mappend = (++)
```

- `[a]`는 타입 a의 값들의 모든 유한 열을 표현
- 연결 `(++)`은 결합법칙을 만족: `(xs ++ ys) ++ zs = xs ++ (ys ++ zs)`
- 빈 리스트는 단위 원소: `[] ++ xs = xs ++ [] = xs`
- 추가 등식 없음: `[a, b] ≠ [b, a]` (교환법칙 없음)

---

## 보편적 구성으로서의 자유 모노이드

자유 모노이드는 보편적 성질로 정의된다:

> 집합 S의 자유 모노이드 F(S)는 모노이드이며, 함수 `η :: S → |F(S)|`(생성원의 포함)이 있어서, 임의의 모노이드 M과 함수 `f :: S → |M|`에 대해 **유일한 모노이드 준동형** `h :: F(S) → M`이 존재하여 `h ∘ η = f`

여기서 `|M|`은 모노이드 M의 **기저 집합(underlying set)** — 모노이드 구조를 잊어버린 집합.

직관: 생성원에 대해 어떤 매핑을 정하면, 자유 모노이드에서의 매핑이 **유일하게** 결정된다.

```haskell
-- 예: 생성원 a를 정수 1로 매핑하면
-- [a] → 합(Sum) 모노이드
-- [a, a, a] ↦ 1 + 1 + 1 = 3
-- 이 매핑은 유일하게 결정됨
```

---

## 망각 펑터 (Forgetful Functor)

**망각 펑터(Forgetful Functor)** U: **Mon** → **Set**은 모노이드의 구조를 "잊어버리고" 기저 집합만 남기는 펑터이다.

- **대상 매핑**: 모노이드 M ↦ 기저 집합 |M|
- **사상 매핑**: 모노이드 준동형 ↦ 대응하는 함수 (구조 보존 조건을 잊음)

반대로, **자유 펑터(Free Functor)** F: **Set** → **Mon**은 집합에서 자유 모노이드를 만든다:

- **대상 매핑**: 집합 S ↦ 자유 모노이드 F(S) = 리스트 [S]
- **사상 매핑**: 함수 f ↦ map f (리스트의 각 원소에 f 적용)

이 둘은 **수반(adjunction)** 관계에 있다: `F ⊣ U`

```
Mon(F(S), M) ≅ Set(S, U(M))
```

"자유 모노이드에서 M으로의 준동형"은 "생성원 집합에서 M의 기저 집합으로의 함수"와 1:1 대응된다.

---

## 자유 구성의 일반적 패턴

자유 모노이드는 더 일반적인 패턴의 특수한 경우다:

| 대수적 구조 | 자유 구성 | Haskell |
|------------|----------|---------|
| 모노이드 | 자유 모노이드 | `[a]` |
| 군(Group) | 자유 군 | 역원을 포함한 열 |
| 카테고리 | 자유 카테고리 | 경로(path)의 모임 |

공통 패턴: 생성원에서 출발하여, 구조의 공리만 만족시키고, 추가 등식은 부과하지 않는다.

---

## 정리

- **자유 모노이드**: 집합의 원소를 생성원으로, 결합법칙과 단위 원소만 부과한 모노이드
- **Haskell에서**: `[a]`가 타입 a의 자유 모노이드. 연산은 `(++)`, 단위 원소는 `[]`
- **보편적 성질**: 생성원에서의 매핑이 모노이드 준동형을 유일하게 결정
- **망각 펑터** U: Mon → Set — 모노이드의 구조를 잊고 집합만 남김
- **자유 펑터** F: Set → Mon — 집합에서 자유 모노이드를 만듦
- **수반 관계**: `F ⊣ U`, 즉 `Mon(F(S), M) ≅ Set(S, U(M))`
- "자유"의 의미: 필수적인 공리 외에 **추가 등식 없음**
