# Chapter 31: Monads, Monoids, and Categories — 모나드, 모노이드, 카테고리

> **핵심 개념**: 이 마지막 장은 책 전체의 핵심 주제들을 하나로 엮는다. 모나드는 자기 함자 카테고리의 모노이드이며, 이 관점을 일반화하면 **이중 카테고리(bicategory)**에서의 모노이드가 된다. 카테고리, 모노이드, 모나드는 서로 다른 추상화 수준에서 같은 구조를 반복한다.

---

## 모노이드의 세 가지 얼굴

모노이드는 카테고리 이론에서 반복적으로 등장하는 구조다:

### 1. 집합으로서의 모노이드

```haskell
class Monoid m where
    mempty  :: m            -- 단위 원소
    mappend :: m -> m -> m  -- 이항 연산 (결합적)
```

### 2. 카테고리로서의 모노이드

대상이 하나인 카테고리. 사상 = 원소, 합성 = 이항 연산, 항등 사상 = 단위 원소.

### 3. 모노이드 카테고리에서의 모노이드 대상

모노이드 카테고리 (C, ⊗, I)에서의 **모노이드 대상** m:
- **곱**: μ :: m ⊗ m → m
- **단위**: η :: I → m
- 결합법칙과 단위법칙 만족

---

## 모나드 = 자기 함자 카테고리의 모노이드

**[C, C]** (자기 함자 카테고리)는 모노이드 카테고리다:
- 텐서 곱 = 펑터 합성 ∘
- 단위 = 항등 펑터 Id

이 모노이드 카테고리에서의 모노이드 대상이 바로 **모나드**:
- 대상: 자기 함자 T
- μ: T ∘ T → T (join)
- η: Id → T (return)

> **"모나드는 자기 함자 카테고리의 모노이드일 뿐이야. 뭐가 문제야?"**
> — Philip Wadler (반쯤 농담)

---

## 이중 카테고리 (Bicategory)

2-카테고리를 약화(weakening)하면 **이중 카테고리(bicategory)**를 얻는다. 2-카테고리에서는 수평 합성의 결합법칙이 엄밀하게 성립하지만, 이중 카테고리에서는 **동형에 이르기까지** 성립한다.

이중 카테고리에서의 모노이드 대상이 모나드의 가장 일반적인 정의를 제공한다.

### Prof — 프로펑터의 이중 카테고리

- **0-셀**: 카테고리
- **1-셀**: 프로펑터 P: C^op × D → Set
- **2-셀**: 자연 변환

프로펑터의 합성은 쌍대끝(coend)으로 정의:

```
(P ∘ Q)(a, c) = ∫^b P(b, c) × Q(a, b)
```

이 이중 카테고리에서의 모노이드가 **화살표(Arrow)**에 가까운 구조를 만든다.

---

## 카테고리의 통합적 관점

| 수준 | 모노이드 | 카테고리 | 연결 |
|------|---------|---------|------|
| 집합 | (M, ·, e) | — | 기본 |
| 카테고리 | 대상 1개인 카테고리 | 대상 여러 개 | 모노이드 ⊂ 카테고리 |
| 2-카테고리 | 모나드 | Cat | 자기 함자의 모노이드 |
| 이중 카테고리 | 일반화된 모나드 | Prof | 프로펑터의 모노이드 |

각 수준에서 같은 패턴이 반복된다:
- **대상과 사상**이 있고
- **합성**이 가능하고 (결합적)
- **항등**이 존재한다

---

## 프로그래밍에서의 의미

### 타입과 함수

```haskell
-- 카테고리 Hask
-- 대상: 타입, 사상: 함수
-- 모노이드 구조: 타입 수준의 곱(×)/합(Either)
```

### 모나드와 효과

```haskell
-- 모나드 = 자기 함자의 모노이드
-- return = 단위, join = 곱, (>>=) = 클라이슬리 합성
-- 효과의 합성이 결합적이고 항등이 있다는 것
```

### 카테고리적 사고

프로그래밍의 핵심은 합성이다. 카테고리 이론은 합성의 수학적 기반을 제공한다:

1. **함수 합성**: 가장 기본적인 형태
2. **펑터 합성**: 구조를 보존하면서 합성
3. **자연 변환의 합성**: 펑터 간의 합성
4. **모나드의 클라이슬리 합성**: 효과를 포함한 합성

이 모든 것이 "대상 + 사상 + 합성 + 항등"이라는 카테고리의 공리를 따른다.

---

## 정리

- **모노이드의 세 얼굴**: 집합의 모노이드, 대상 1개인 카테고리, 모노이드 카테고리의 모노이드 대상
- **모나드 = [C,C]의 모노이드**: 자기 함자 카테고리에서 펑터 합성을 텐서 곱으로
- **이중 카테고리**: 2-카테고리의 약화. 합성의 결합법칙이 동형에 이르기까지 성립
- **Prof**: 프로펑터의 이중 카테고리. 합성은 쌍대끝으로 정의
- **반복되는 패턴**: 대상 + 사상 + 합성 + 항등 — 모든 추상화 수준에서 동일
- **프로그래밍의 핵심 = 합성**: 함수, 펑터, 자연 변환, 모나드 — 모두 카테고리적 합성
- **"카테고리 이론은 합성에 대한 이론이다"** — 이 책의 첫 장에서 시작한 주제로 돌아옴
