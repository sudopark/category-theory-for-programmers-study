# Chapter 19: Free/Forgetful Adjunctions — 자유/망각 수반

> **핵심 개념**: **자유 펑터(Free Functor)**는 **망각 펑터(Forgetful Functor)**의 좌수반이다. 이 수반은 "구조를 잊는 것"과 "최소한의 구조를 만드는 것" 사이의 깊은 관계를 표현한다. 자유 모노이드(리스트), 자유 카테고리(경로) 등이 대표적 예시다.

---

## 망각 펑터 (Forgetful Functor)

**망각 펑터** U: C → Set은 대수적 구조를 "잊어버리고" 기저 집합만 남기는 펑터:

- U: **Mon** → **Set**: 모노이드 → 기저 집합 (이항 연산과 단위 원소를 잊음)
- U: **Grp** → **Set**: 군 → 기저 집합 (연산과 역원을 잊음)
- U: **Cat** → **Graph**: 카테고리 → 그래프 (합성 규칙을 잊음)

```haskell
-- 모노이드의 망각: (String, (++), "") → String의 값들의 집합
-- 구조(연산)를 잊고, 값만 남긴다
```

---

## 자유 펑터 (Free Functor)

**자유 펑터** F: Set → C는 집합에서 "최소한의 공리만 부과하여" 대수적 구조를 만드는 펑터:

```
F ⊣ U    -- 자유 펑터는 망각 펑터의 좌수반
```

수반의 홈셋 동형:

```
C(F(S), M) ≅ Set(S, U(M))
```

**"자유 구조에서 M으로의 구조 보존 사상"은 "생성원에서 M의 기저 집합으로의 함수"와 1:1 대응된다."**

---

## 구체적 예시

### 자유 모노이드

```
Mon(F(S), M) ≅ Set(S, U(M))
```

- F(S) = `[S]` (S의 원소들의 리스트)
- 생성원에서의 매핑 `f :: S → U(M)`을 정하면
- 모노이드 준동형 `h :: [S] → M`이 유일하게 결정: `h [s1, s2, ...] = f(s1) `mappend` f(s2) `mappend` ...`

```haskell
-- 예: 생성원 {a, b}를 정수 합 모노이드로 매핑
-- f a = 2, f b = 3이면
-- h [a, b, a] = 2 + 3 + 2 = 7
-- 이 h는 유일하게 결정됨!
```

### 단위와 쌍대단위

**단위** η: S → U(F(S)) = S → [S]:

```haskell
eta :: a -> [a]
eta x = [x]    -- 원소를 싱글톤 리스트로 포함
```

**쌍대단위** ε: F(U(M)) → M:

리스트의 각 원소를 모노이드 연산으로 결합:

```haskell
epsilon :: Monoid m => [m] -> m
epsilon = mconcat    -- 리스트의 원소들을 모노이드 연산으로 접기
```

## 자유 구성의 직관

자유 구성은 **"추가 등식 없이"** 구조를 만든다:

| 구조 | 잊는 것 | 자유 구성 | Haskell |
|------|---------|----------|---------|
| 모노이드 | 연산, 단위 | 자유 모노이드 | `[a]` |
| 군 | 연산, 역원, 단위 | 자유 군 | 역원 포함 열 |
| 카테고리 | 합성 규칙 | 자유 카테고리 | 경로 |

**자유 = 필수 공리만 만족 + 추가 등식 없음**

예를 들어 자유 모노이드에서 `[a, b] ≠ [b, a]` — 교환법칙 같은 추가 등식을 부과하지 않았기 때문이다.

---

## 수반의 관점에서

자유-망각 수반의 핵심 메시지:

1. **구조를 잊는 것(U)**은 쉽다 — 정보를 버리기만 하면 됨
2. **구조를 만드는 것(F)**은 가장 경제적인 방법이 있다 — 자유 구성
3. 이 둘은 **수반으로 연결**된다: F가 U의 "최적의 왼쪽 역"

자유 구성이 "최적"인 이유: 생성원에서의 매핑만으로 전체 준동형이 결정되므로, 불필요한 등식이 없는 가장 큰(자유로운) 구조다.

---

## Haskell에서의 자유 모나드

자유-망각 수반의 패턴은 **자유 모나드(Free Monad)**로도 확장된다:

```haskell
data Free f a = Pure a | Free (f (Free f a))

-- Free f는 펑터 f에 대한 자유 모나드
-- Pure = return (단위)
-- Free = 한 단계의 효과를 쌓음
```

자유 모나드는 "효과의 구문 트리"를 만들고, 해석기(interpreter)가 나중에 의미를 부여한다. 이것이 자유 구성의 프로그래밍적 응용이다.

---

## 정리

- **망각 펑터** U: 대수적 구조를 잊고 기저 집합만 남김
- **자유 펑터** F: 집합에서 최소 공리만으로 대수적 구조를 만듦
- **수반**: `F ⊣ U`, 즉 `C(F(S), M) ≅ Set(S, U(M))`
- **단위** η: 생성원을 자유 구조에 포함. **쌍대단위** ε: 자유 구조를 대수 연산으로 접기
- **자유 모노이드** = 리스트 `[a]`. 생성원 매핑 → 준동형 유일 결정
- **"자유"** = 필수 공리만 부과, 추가 등식 없음
- **자유 모나드**: 효과의 구문 트리를 만들고 나중에 해석하는 패턴
