# Chapter 10: Natural Transformations — 자연 변환

> **핵심 개념**: **자연 변환(Natural Transformation)**은 펑터 사이의 사상이다. 두 펑터 F, G: C → D가 있을 때, 자연 변환은 F의 상을 G의 상으로 매핑하되 **자연성 조건(naturality condition)**을 만족한다. 프로그래밍에서 **다형 함수(polymorphic function)**가 바로 자연 변환이다.

---

## 자연 변환이란?

같은 두 카테고리 C, D 사이에 여러 펑터가 존재할 수 있다. 펑터는 카테고리를 다른 카테고리에 "임베딩"하는 방법인데, 같은 소스를 다르게 임베딩하는 방법들 사이의 관계를 이해하고 싶다.

**자연 변환 α: F → G**는 C의 **각 대상 a**에 대해 D의 사상을 하나씩 지정한다:

```
α_a :: F a → G a   -- a에서의 "성분(component)"
```

이 성분들은 아무렇게나 골라도 되는 게 아니라, **자연성 조건**을 만족해야 한다.

---

## 자연성 조건 (Naturality Condition)

C의 임의의 사상 `f :: a → b`에 대해, 다음 다이어그램이 **가환(commute)**해야 한다:

```
      α_a
F a ------→ G a
 |            |
 | F f        | G f
 ↓            ↓
F b ------→ G b
      α_b
```

즉: **`G f ∘ α_a = α_b ∘ F f`**

직관: "먼저 변환(α)하고 나서 매핑(Gf)해도, 먼저 매핑(Ff)하고 나서 변환(αb)해도 같은 결과"

이것은 자연 변환이 단순히 각 대상에서 독립적으로 작동하는 것이 아니라, 카테고리의 구조와 **조화롭게(coherently)** 작동한다는 뜻이다.

---

## 다형 함수 = 자연 변환

프로그래밍에서 자연 변환은 **매개변수적 다형 함수(parametrically polymorphic function)**로 나타난다.

### 예시 1: safeHead

```haskell
safeHead :: [a] -> Maybe a
safeHead []    = Nothing
safeHead (x:_) = Just x
```

이것은 List 펑터에서 Maybe 펑터로의 자연 변환이다. 자연성 조건을 확인해보자:

```haskell
-- fmap f . safeHead = safeHead . fmap f  가 성립하는가?

-- 빈 리스트:
fmap f (safeHead []) = fmap f Nothing = Nothing
safeHead (fmap f []) = safeHead [] = Nothing  ✓

-- 비지 않은 리스트:
fmap f (safeHead (x:xs)) = fmap f (Just x) = Just (f x)
safeHead (fmap f (x:xs)) = safeHead (f x : fmap f xs) = Just (f x)  ✓
```

### 예시 2: length

```haskell
length :: [a] -> Int    -- List 펑터에서 Const Int 펑터로의 자연 변환
```

### 예시 3: 다양한 자연 변환들

```haskell
-- Maybe에서 List로
maybeToList :: Maybe a -> [a]
maybeToList Nothing  = []
maybeToList (Just x) = [x]

-- Reader 펑터에서 값 추출
dumb :: (->) r a -> a    -- 실제로는 (r -> a) -> a, r 값이 필요하므로 불가
                          -- 이런 자연 변환은 존재하지 않을 수 있다
```

### "무료 정리" (Free Theorems)

매개변수적 다형성(parametric polymorphism)에서 자연성 조건은 **자동으로 만족**된다. 이것이 Wadler의 "무료 정리(theorems for free)"다.

타입만 보고도 함수의 행동에 대한 정리를 공짜로 얻을 수 있다:
- `f :: [a] -> [a]`인 함수는 반드시 `fmap g . f = f . fmap g`를 만족
- 원소의 내용을 들여다볼 수 없으므로 구조적 조작만 가능

## 펑터 카테고리 (Functor Category)

카테고리 C에서 D로의 모든 펑터와, 그들 사이의 모든 자연 변환은 새로운 카테고리를 이룬다:

- **대상**: C에서 D로의 펑터들
- **사상**: 자연 변환들
- **합성**: 자연 변환의 (수직) 합성
- **항등**: 각 펑터의 항등 자연 변환

이 카테고리를 **펑터 카테고리** **[C, D]** 또는 **D^C**로 쓴다.

### 자연 변환의 합성

자연 변환 `α :: F → G`와 `β :: G → H`의 합성 `β ∘ α :: F → H`:

```
(β ∘ α)_a = β_a ∘ α_a    -- 각 성분에서 합성
```

항등 자연 변환의 각 성분은 항등 사상 `id_{Fa}`이다.

---

## 2-카테고리 (2-Category)

**Cat**(작은 카테고리들의 카테고리)에서 대상은 카테고리이고 사상은 펑터다. 그런데 펑터 사이에 자연 변환이라는 "사상의 사상"이 있다. 이처럼 두 수준의 사상이 있는 구조를 **2-카테고리**라 한다.

Cat에서:
- **0-셀(대상)**: 카테고리
- **1-셀(사상)**: 펑터
- **2-셀(사상의 사상)**: 자연 변환

### 수직 합성 (Vertical Composition)

같은 두 카테고리 사이의 자연 변환을 합성하는 것. 위에서 본 일반적인 합성:

```
α :: F → G, β :: G → H  ⟹  β ∘ α :: F → H
```

### 수평 합성 (Horizontal Composition)

서로 다른 카테고리 쌍을 잇는 펑터와 자연 변환을 합성하는 것.

펑터 F, G :: C → D와 F', G' :: D → E가 있고, α :: F → G, β :: F' → G'일 때:

```
β ∘ₕ α :: F' ∘ F → G' ∘ G
```

각 성분에서:
```
(β ∘ₕ α)_a = β_{Ga} ∘ F'(α_a) = G'(α_a) ∘ β_{Fa}
```

이 두 가지가 같다는 것이 바로 β의 자연성 조건에서 나온다.

수평 합성은 **위스커링(whiskering)**이라는 특수한 경우도 포함한다:
- 펑터와 자연 변환의 합성 (한쪽이 항등 자연 변환)

---

## 정리

- **자연 변환**: 펑터 간의 사상. 각 대상에서의 성분 `α_a :: F a → G a`로 구성
- **자연성 조건**: `G f ∘ α_a = α_b ∘ F f` — "변환 후 매핑 = 매핑 후 변환"
- **다형 함수 = 자연 변환**: 매개변수적 다형성에서 자연성은 자동으로 보장 ("무료 정리")
- **펑터 카테고리 [C, D]**: 대상=펑터, 사상=자연 변환
- **2-카테고리 Cat**: 대상=카테고리, 1-셀=펑터, 2-셀=자연 변환
- **수직 합성**: 같은 카테고리 쌍 사이의 자연 변환 합성
- **수평 합성**: 서로 다른 카테고리 쌍을 잇는 자연 변환 합성
