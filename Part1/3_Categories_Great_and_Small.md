# Chapter 3: Categories Great and Small — 크고 작은 카테고리들

> **핵심 개념**: 카테고리는 집합의 카테고리(Set) 외에도 무수히 많다. 대상이 없는 빈 카테고리, 순서 관계를 나타내는 카테고리, 그리고 하나의 대상만 가지는 **모노이드(Monoid)** 카테고리 등 다양한 예시를 통해 카테고리의 일반성을 이해한다.

---

## 빈 카테고리 (Empty Category)

가장 단순한 카테고리: 대상도 없고 사상도 없다.

- 대상이 없으므로 사상도 없다
- 카테고리의 공리(합성, 결합법칙, 항등 사상)는 검증할 대상 자체가 없으므로 **자동으로 만족**(공허하게 참, vacuously true)
- 그 자체로는 쓸모없어 보이지만, 카테고리들의 카테고리 같은 더 큰 맥락에서 의미를 가진다 (숫자 0처럼)

---

## 단일 대상 카테고리

하나의 대상만 존재하는 카테고리. 최소한 하나의 사상(항등 사상)이 반드시 있어야 한다. 이것 역시 더 큰 맥락에서 중요한 역할을 한다.

---

## 자유 카테고리 (Free Category)

방향 그래프(directed graph)가 주어지면, 카테고리의 공리를 만족시키기 위해 필요한 사상들을 추가하여 카테고리를 만들 수 있다:

1. 각 노드(대상)에 항등 사상 추가
2. 합성 가능한 모든 화살표 쌍에 대해 합성 사상 추가
3. 결합법칙을 만족하도록 계속 추가

이렇게 그래프에서 **최소한의 사상만 추가하여** 만든 카테고리를 **자유 카테고리(free category)**라 한다. "자유"란 카테고리의 공리 외에 추가 제약이 없다는 뜻이다.

### 예시: 그래프에서 자유 카테고리 만들기

원래 그래프:
```
A --f--> B --g--> C
```

자유 카테고리로 완성:
```
A --f--> B --g--> C
|        |        |
id_A     id_B     id_C    ← 항등 사상 추가
|                 ↑
+--- g∘f ---------+       ← 합성 사상 추가
```

- 항등 사상 3개 추가: `id_A`, `id_B`, `id_C`
- 합성 사상 1개 추가: `g ∘ f : A → C`
- 결합법칙은 자동 만족 (3단 이상 합성이 없으므로)

이것이 이 그래프에서 만들 수 있는 **가장 작은 카테고리**다. 여기에 "f ∘ g = id" 같은 추가 등식을 붙이지 않았기 때문에 "자유"롭다.

---

## 홈셋 (Hom-set)

카테고리 C에서 대상 a에서 b로 향하는 **모든 사상의 집합**을 **홈셋(hom-set)**이라 하고, `C(a, b)` 또는 `Hom_C(a, b)`로 쓴다.

- Set 카테고리에서: `Set(Int, String)` = Int에서 String으로 가는 모든 함수의 집합 → 매우 클 수 있다
- 순서 카테고리에서: 홈셋은 공집합(화살표 없음)이거나 singleton(화살표 하나) → 매우 작다

홈셋의 크기가 카테고리의 성격을 결정한다. 이 개념은 이후 순서 카테고리, 모노이드 카테고리에서 반복적으로 등장한다.

---

## 순서 카테고리 (Orders)

"작거나 같다(≤)" 같은 순서 관계가 카테고리를 이루는지 확인해보자:

- **항등 사상**: 모든 대상은 자기 자신 이하다 (a ≤ a) — 반사성 ✓
- **합성**: a ≤ b이고 b ≤ c이면 a ≤ c — 추이성 ✓
- **결합법칙**: 자연스럽게 만족 ✓

이 두 조건(반사성 + 추이성)이 순서 카테고리의 공통 베이스이며, 여기에 조건을 얹어가며 원순서 → 부분순서 → 전순서로 강화된다.

포함 관계: `전순서 ⊂ 부분순서 ⊂ 원순서`

### 원순서 (Preorder) — 순수하게 카테고리적

반사성 + 추이성만 요구한다. **대상의 동일성을 따지지 않는**, 가장 순수한 카테고리적 순서 개념이다.

- `a ≤ b`이면서 동시에 `b ≤ a`일 수 있다 (a ≠ b여도) — 양방향 화살표 허용
- 두 대상 사이의 사상은 최대 하나이므로, 원순서는 곧 **얇은 카테고리(thin category)**다
- 모든 홈셋이 공집합이거나 singleton인 카테고리 = 얇은 카테고리

### 부분 순서 (Partial Order) — 원순서 + 반대칭성

원순서에 **반대칭성(antisymmetry)**을 추가: a ≤ b이고 b ≤ a이면 반드시 a = b.

- 여기서 처음으로 **대상의 동일성**을 따진다 — 순수 카테고리 조건을 넘어선 집합론적 조건이 추가된 것
- 서로 다른 두 대상 사이에 양방향 화살표가 동시에 존재하지 않는다
- 비교 불가능한 쌍은 강요하지 않고 그냥 둔다

> **예시**: 집합의 포함관계 ⊆ — `{1,2}`와 `{2,3}`은 서로 포함 관계가 없으므로 비교 불가능

### 전순서 (Total Order) — 부분순서 + 전체성

부분 순서에서 **어떤 두 대상이든 비교 가능**한 관계. 임의의 a, b에 대해 a ≤ b이거나 b ≤ a이다.

- 비교 불가능한 쌍을 허용하지 않는다

> **예시**: 정수의 ≤ — 어떤 두 정수를 골라도 반드시 크거나 같거나 작다

### 핵심 구분

| | 추가 조건 | 대상의 동일성을 따지나? |
|---|---|---|
| **원순서** | 없음 (반사성 + 추이성만) | ❌ 순수 카테고리 |
| **부분순서** | 양방향 화살표 → 동일 대상 | ✅ |
| **전순서** | 모든 쌍 비교 가능 | ✅ |

- **원순서 → 부분순서**: "서로 왔다갔다 할 수 있으면 사실 같은 대상이다" 규칙 추가
- **부분순서 → 전순서**: "비교 못 하는 쌍이 없어야 한다" 규칙 추가

## 집합으로서의 모노이드 (Monoid as Set)

**모노이드(Monoid)**는 다음을 갖춘 집합이다:

1. **이항 연산**: 결합법칙을 만족하는 연산 (`a ⊕ (b ⊕ c) = (a ⊕ b) ⊕ c`)
2. **단위 원소(unit element)**: 연산의 항등원 (`e ⊕ a = a ⊕ e = a`)

예시:
- 자연수 + 덧셈, 단위 원소 = 0
- 자연수 + 곱셈, 단위 원소 = 1
- 문자열 + 연결(concatenation), 단위 원소 = "" (빈 문자열)

```haskell
class Monoid m where
    mempty  :: m            -- 단위 원소
    mappend :: m -> m -> m  -- 이항 연산

instance Monoid String where
    mempty  = ""
    mappend = (++)
```

```swift
protocol Monoid {
    static var mempty: Self { get }
    func mappend(_ other: Self) -> Self
}

extension String: Monoid {
    static var mempty: String { "" }
    func mappend(_ other: String) -> String { self + other }
}
```

> `mappend`의 시그니처 `m -> m -> m`은 커링에 의해 `m -> (m -> m)`으로 읽을 수 있다: 모노이드의 원소를 받아 **m에서 m으로의 함수**를 반환한다. 이 관점이 곧 카테고리적 해석으로 이어진다.

## 카테고리로서의 모노이드 (Monoid as Category)

### 핵심 관점 전환

집합으로서의 모노이드는 **"원소들의 집합 + 연산"**이었다면, 카테고리로서의 모노이드는 **"원소를 잊고, 원소가 하는 행동만 본다"**는 것이다.

```
집합 관점:     원소 5가 있다
카테고리 관점:  "5를 더하는 행동(adder 5)"이 있다
```

원소 자체가 아니라 **원소가 유발하는 변환**에 집중하는 것이 핵심이다.

### adder 예시로 구체화

자연수 덧셈 모노이드를 카테고리로 표현하면:

```
대상: * (단 하나)

사상들:
  adder 0: * → *   (항등 사상, 아무것도 안 함)
  adder 1: * → *
  adder 2: * → *
  adder 5: * → *
  ...

합성:
  adder 5 ∘ adder 3 = adder 8   (5 + 3 = 8 과 동일)
```

모든 사상이 `* → *`이므로 **항상 합성 가능**하고, 합성의 결과도 `* → *`이므로 **닫혀 있다.**

### 대응 관계

| 집합으로서의 모노이드 | 카테고리로서의 모노이드 |
|---|---|
| 모노이드 집합 M | 단 하나의 대상 `*` |
| 원소 `a ∈ M` | 사상 `a: * → *` |
| 이항 연산 `a ⊕ b` | 사상의 합성 `b ∘ a` |
| 단위 원소 `e` | 항등 사상 `id` |

모노이드 집합 M의 원소들은 카테고리 모노이드의 **홈셋 `M(*, *)`**의 원소에 1:1 대응된다.

### 왜 대상이 하나인가?

집합 모노이드에서 이항 연산 `m ⊕ m → m`은 **항상 같은 타입(m)에서 같은 타입(m)으로** 가는 연산이다. 카테고리 언어로 하면 출발지와 도착지가 항상 같은 대상이므로, 대상이 하나만 있으면 충분하다. 모든 흥미로운 구조는 그 대상 위의 사상들(= 홈셋 `M(*, *)`)에 담긴다.

### `mappend` 커링과의 연결

> `mappend :: m -> m -> m`을 커링하면 `m -> (m -> m)`으로 읽힌다.

이것은 정확히:
- 모노이드 원소 하나를 받아서
- `* → *`인 사상(함수) 하나를 돌려준다

즉 **"원소를 사상으로 매핑"**하는 것이 `mappend`의 카테고리적 의미다. 이 시그니처 자체가 "집합의 원소 → 카테고리의 화살표" 전환을 표현하고 있다.

## 정리

- **빈 카테고리**: 대상도 사상도 없는 가장 단순한 카테고리
- **자유 카테고리**: 그래프에서 카테고리 공리를 만족하도록 최소한의 사상을 추가하여 생성
- **순서**는 카테고리다: 원순서(preorder), 부분 순서(partial order), 전순서(total order)
- **얇은 카테고리**: 홈셋이 공집합이거나 singleton — 원순서와 1:1 대응
- **홈셋** `C(a, b)`: 대상 a에서 b로의 모든 사상의 집합
- **모노이드 = 하나의 대상만 가지는 카테고리**: 원소 → 사상, 이항 연산 → 합성, 단위 원소 → 항등 사상
