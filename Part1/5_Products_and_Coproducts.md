# Chapter 5: Products and Coproducts — 곱과 쌍대곱

> **핵심 개념**: 카테고리에서 특정 대상을 특징짓는 방법이 **보편적 구성(universal construction)**이다. 이 방법으로 **시작 대상**, **끝 대상**, **곱(product)**, **쌍대곱(coproduct)**을 정의한다. 화살표를 뒤집으면 쌍대 개념을 얻는 **쌍대성(duality)**도 핵심이다.

---

## 보편적 구성 (Universal Construction)

카테고리에서 대상의 내부를 들여다볼 수 없으므로, 특정 대상을 다른 대상과의 **관계 패턴**으로 특징짓는다. 이 과정은 구글 검색과 비슷하다:

1. **패턴 정의**: 대상과 사상으로 이루어진 패턴을 설정
2. **후보 검색**: 카테고리에서 이 패턴에 맞는 모든 후보를 찾음 (보통 많다)
3. **랭킹**: 후보들 사이에 순위를 매김 (사상을 이용)
4. **최적 선택**: 가장 "좋은" 후보를 선택 — 이것이 보편적 구성의 결과

---

## 시작 대상 (Initial Object)

가장 단순한 패턴: 대상 하나. 수많은 후보 중 "가장 처음"에 해당하는 대상을 찾자.

랭킹: 대상 a에서 b로의 사상이 있으면 a가 b보다 "더 initial"하다.

> **정의**: 카테고리에서 **시작 대상(initial object)**은 모든 대상으로 가는 **유일한** 사상이 존재하는 대상이다.

- Set 카테고리에서 시작 대상은 **공집합(empty set)** = Haskell의 `Void`
- `absurd :: Void -> a` — 공집합에서 모든 타입으로의 유일한 함수

```haskell
absurd :: Void -> a
```

시작 대상은 (존재한다면) **동형사상에 이르기까지 유일**하다.

---

## 끝 대상 (Terminal Object)

시작 대상의 반대: 화살표 방향을 뒤집는다.

> **정의**: 카테고리에서 **끝 대상(terminal object)**은 모든 대상에서 오는 **유일한** 사상이 존재하는 대상이다.

- Set 카테고리에서 끝 대상은 **단원소 집합(singleton)** = Haskell의 `()`

```haskell
unit :: a -> ()
unit _ = ()
```

**유일성이 핵심이다.** Bool로 향하는 함수는 `yes _ = True`와 `no _ = False` 두 개가 있으므로 Bool은 끝 대상이 아니다. Unit으로 향하는 함수는 `unit _ = ()`뿐이므로 끝 대상이다.

---

## 쌍대성 (Duality)

시작 대상과 끝 대상의 정의는 화살표 방향만 다르다. 이것이 **쌍대성(duality)**의 핵심이다.

어떤 카테고리 **C**의 모든 화살표를 뒤집으면 **반대 카테고리(opposite category)** **C^op**를 얻는다:

```haskell
-- 원래 카테고리
f :: a -> b
g :: b -> c
h = g . f :: a -> c

-- 반대 카테고리
f_op :: b -> a
g_op :: c -> b
h_op = f_op . g_op :: c -> a  -- 합성 순서도 뒤집힘
```

쌍대성 덕분에 어떤 구성을 증명하면 그 쌍대(co-)도 자동으로 얻는다. 접두사 **co-**를 붙여 부른다:
- product ↔ **co**product
- monad ↔ **co**monad  
- cone ↔ **co**cone
- limit ↔ **co**limit

> 화살표를 두 번 뒤집으면 원래로 돌아오므로 co-co-monad 같은 것은 없다.

## 동형사상 (Isomorphism)

두 대상이 "같다"는 것을 카테고리적으로 어떻게 정의할까? 엄밀한 동등(equality)은 너무 강한 조건이다. 대신 **동형(isomorphism)**을 사용한다.

사상 `f :: a → b`가 동형사상이려면, 역사상 `g :: b → a`가 존재하여:

```haskell
g . f = id_a
f . g = id_b
```

직관적으로, 동형인 대상들은 "같은 모양"이다 — 한 대상의 각 부분이 다른 대상의 각 부분에 1:1로 대응된다.

### 시작 대상의 유일성 증명

두 시작 대상 i₁, i₂가 있다고 하자:
- i₁은 시작 대상이므로 `f :: i₁ → i₂`가 **유일**
- i₂도 시작 대상이므로 `g :: i₂ → i₁`이 **유일**
- `g . f :: i₁ → i₁`인데, i₁에서 i₁로의 사상은 `id`뿐 → `g . f = id`
- 마찬가지로 `f . g = id`
- 따라서 i₁ ≅ i₂ (동형)

사상의 **유일성** 덕분에 동형사상 자체도 유일하다. 이 "유일한 동형에 이르는 유일성"은 모든 보편적 구성의 핵심 성질이다.

## 곱 (Product)

두 집합의 **곱집합(Cartesian product)**은 쌍(pair)의 집합이다. 곱집합과 원래 집합을 연결하는 패턴은 **투영(projection)** 함수다:

```haskell
fst :: (a, b) -> a
fst (x, _) = x

snd :: (a, b) -> b
snd (_, y) = y
```

### 보편적 구성으로 곱 정의하기

**패턴**: 대상 c와 두 개의 투영 사상 `p :: c → a`, `q :: c → b`

이 패턴을 만족하는 c는 많다. 예를 들어 `Int`와 `Bool`의 곱을 찾는다면:

```haskell
-- 후보 1: Int (정보가 부족)
p :: Int -> Int
p x = x
q :: Int -> Bool
q _ = True          -- Bool 차원의 정보를 잃어버림

-- 후보 2: (Int, Int, Bool) (정보가 과다)
p :: (Int, Int, Bool) -> Int
p (x, _, _) = x
q :: (Int, Int, Bool) -> Bool
q (_, _, b) = b     -- 두 번째 Int는 불필요한 정보
```

**랭킹**: 후보 c'가 c보다 "못하다"는 것은, `m :: c' → c`가 존재하여 c'의 투영이 c의 투영으로 분해(factorize)될 수 있다는 것이다:

```haskell
p' = p . m
q' = q . m
```

**최적의 곱**: 다른 모든 후보를 분해할 수 있는 **가장 좋은** 대상. 다른 모든 후보 c'에서 곱 c로의 **유일한** 사상 m이 존재:

```haskell
m :: c -> (a, b)
m x = (p x, q x)
```

> **곱의 정의**: 두 대상 a, b의 곱은 투영 p, q를 갖춘 대상 c로, 임의의 다른 후보 c'에 대해 그것의 투영을 분해하는 **유일한 사상** m이 존재한다.

투영을 분해하는 고차 함수를 **factorizer**라 부른다:

```haskell
factorizer :: (c -> a) -> (c -> b) -> (c -> (a, b))
factorizer p q = \x -> (p x, q x)
```

## 쌍대곱 (Coproduct)

곱의 **쌍대**: 화살표를 뒤집는다. 투영(projection) 대신 **주입(injection)**:

```haskell
i :: a -> c
j :: b -> c
```

> **쌍대곱의 정의**: 두 대상 a, b의 쌍대곱은 주입 i, j를 갖춘 대상 c로, 임의의 다른 후보 c'에 대해 `m :: c → c'`가 존재하여 c의 주입을 분해한다.

```haskell
i' = m . i
j' = m . j
```

Set 카테고리에서 쌍대곱은 **분리 합집합(disjoint union)**이다. Haskell에서는 `Either`로 구현된다:

```haskell
data Either a b = Left a | Right b
```

```swift
enum Either<A, B> {
    case left(A)
    case right(B)
}
```

`Left`와 `Right`가 각각 주입 함수 역할을 한다.

쌍대곱의 factorizer:

```haskell
factorizer :: (a -> c) -> (b -> c) -> Either a b -> c
factorizer i j (Left a)  = i a
factorizer i j (Right b) = j b
```

---

## 비대칭성 (Asymmetry)

쌍대 개념들(시작↔끝, 곱↔쌍대곱)은 정의가 대칭적이지만, Set 카테고리에서의 구현은 매우 다르다:

| 개념 | 역할 | Set에서의 구현 |
|------|------|---------------|
| 시작 대상 | 0처럼 작동 | 공집합 `Void` |
| 끝 대상 | 1처럼 작동 | 단원소 집합 `()` |
| 쌍대곱 | 덧셈처럼 작동 | `Either a b` |
| 곱 | 곱셈처럼 작동 | `(a, b)` |

이 비대칭성은 함수 자체의 비대칭성에서 비롯된다:
- 함수는 정의역의 **모든** 원소에 대해 정의되어야 하지만, 공역을 다 채울 필요는 없다
- 정의역의 여러 원소가 공역의 같은 원소로 갈 수 있다 (surjective가 아닐 수 있음)
- 정의역의 서로 다른 원소가 합쳐질 수 있다 (injective가 아닐 수 있음)

---

## 정리

- **보편적 구성**: 패턴 정의 → 후보 검색 → 랭킹 → 최적 선택
- **시작 대상**: 모든 대상으로 가는 유일한 사상이 있는 대상 (Set에서 `Void`)
- **끝 대상**: 모든 대상에서 오는 유일한 사상이 있는 대상 (Set에서 `()`)
- **쌍대성**: 화살표를 뒤집으면 쌍대 개념을 얻는다 (co- 접두사)
- **동형사상**: 역사상이 있는 사상. 보편적 구성의 결과는 동형에 이르기까지 유일
- **곱**: 두 투영을 갖춘 대상의 보편적 구성 (Set에서 쌍 `(a, b)`)
- **쌍대곱**: 두 주입을 갖춘 대상의 보편적 구성 (Set에서 `Either a b`)
- **factorizer**: 보편적 구성에서 유일한 사상 m을 만들어내는 고차 함수
